# Main Dashboard — Software Requirements Specification (SRS)

**Version:** 2.0 (rebuilt from master spreadsheets)
**Date:** 2026-06-05
**Owner:** Peter (AI COO)
**Status:** Alpha — verified, deployed; pending Jonathan/Brad UAT
**Spec source:** Jonathan's finalized dashboard/pipeline notes (6/5), captured in
`memories/research/alpha-feedback-jonathan-june2026.md`

---

## 1. Purpose

Give the owners a single at-a-glance view of the business, driven entirely by the
live master spreadsheets so every number matches what they already trust. Replaces
the prior dashboard, which derived KPIs from an older bid-log-only data path and
could disagree with the masters.

## 2. Data source (single source of truth)

All values come from `window.PF_PROJECTS`, generated by
`platform/sync/extract-projects.py` from:
- **PF Project Master.xlsx** → tab "2026 WIP & Completed Projects"
- **Project Bid Log.xlsx** → tab "Agg Pier Bid Log"

KPIs are computed in Python (one place), so the frontend only renders. This
eliminates the "same KPI, different formula per module" defect class.

## 3. Terminology (FINAL — Jonathan, 6/5)

| Term | Definition |
|------|-----------|
| **Outstanding Bids** | Bids submitted / in-flight (status `Submitted` or `Submitted - Edging`). Headline shows **live** bids only (decision due within last 90 days) to exclude stale entries never closed out; full count/value retained as `outstandingAll*`. |
| **Pipeline Value** | Awarded projects not yet completed (the WIP tab). |
| **FY26 Revenue** | Completed projects in the FY (calendar year Jan–Dec). |
| **Total FY26 Sales to Date** | FY26 Revenue + FY26 Pipeline Value. |
| **Upcoming Bids** | Bids with a due date within the next 7 days (any status). |
| **Recent bids** | Outstanding (live submitted) bids only — no projects we did not bid. |

## 4. Functional requirements

1. **Top summary bar** — Total FY26 Sales to date, with Revenue + Pipeline
   breakdown and progress vs the $5.25M FY26 goal.
2. **KPI tiles** — Outstanding Bids (count + value), Pipeline Value (count),
   FY26 Revenue (% of goal), AR Outstanding (+ retainage due).
3. **Upcoming Bids** table — projects due ≤7 days (Project, GC, Due, Status).
4. **Active Projects** table — in-field jobs with % progress.
5. **Recent Bids** table — live submitted bids (Project, GC, Status, Due, Value);
   subtitle shows "N live of M total".
6. **Upcoming Projects** table — awarded, not yet started.
7. **Completed Projects (FY26)** table — subtitle shows count + total value.
8. All tables show an explicit empty state when no rows.

## 5. Non-functional requirements

- **Security:** every spreadsheet-sourced string rendered via `E()`→`window.esc()`
  (fail-closed if `esc` missing). Data generators escape `</`, U+2028/9, `<!--`
  so spreadsheet text cannot break out of a `<script>`.
- **Robustness:** missing `PF_PROJECTS` shows a clear "data unavailable" note;
  undefined KPI fields coerce to 0 (never NaN).
- **Accuracy:** every KPI must tie to the master tab's own roll-up rows.
