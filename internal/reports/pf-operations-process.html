Estimating SOP

Aggregate Pier estimating process from bid invitation through submission

1
Receive Bid Invitation
Open GC link, download geotech report, and confirm project requires aggregate piers.
  • Read geotech report to confirm AP is required
  • Run geotech through AI if time allows for secondary confirmation
  • If no AP needed, pass on the project
Estimating
2
Download Drawings
Collect all relevant project documents from the GC's bid invitation.
  • Civil Drawings
  • Structural Drawings
  • Architectural Drawings
  • Schedules, specifications, bid forms, scopes of work
Estimating
3
Create Project in Bid Log
Add project to the SharePoint Bid Log with all pertinent information for tracking.
  • Due date, contact info, project details
  • Location: SharePoint > 03 - Estimating > Bid Log
  • Ensure all PF team members can track quickly
Estimating
4
Set Up Project Folder
Create structured project folder in SharePoint under Actively Bidding.
  • Create folder: SharePoint > 03 - Estimating > 01 - Aggregate Piers > 20XX Project Bids > 01 - Actively Bidding
  • Subfolder structure: Drawings / Specs / Geotech / Addenda / Other Bid Docs / Vendor Quotes
  • Copy Word Template Proposal and Excel Template Estimate into root of project folder
  • Rename: "YY-MMDD Project Name - PF Bid" and "YY-MMDD Project Name Estimate"
Estimating
5
Package Bid Docs
Extract relevant drawing pages in Bluebeam to create a focused bid document for Dr. Ed.
  • Architectural: Title page, lowest/first floor plans, site plan
  • Civil: Alta survey, demo plan, site plan, grading plans
  • Structural: Note pages (S001-S005), foundation plans (S100), concrete details + schedules (S300-S500)
  • Save as "Project Name - PF Bid Docs"
  • Reduce file size in Bluebeam if over 25MB
Estimating
6
Send to Dr. Ed
Email Dr. Ed Garbin with the geotech report, bid docs, and specs requesting preliminary design.
  • Send: Geotech report + "Project Name - PF Bid Docs" + specs (if applicable)
  • Minimum 5 working days turnaround; 2+ weeks preferred
  • If addenda received, send relevant updates to Dr. Ed ASAP
  • Pair down addenda to AP-relevant content only
Dr. Ed Garbin / Estimating
7
Do Takeoffs
Perform quantity takeoffs in Kreo.com while Dr. Ed works on the preliminary design.
  • Platform: Kreo.com (takeoff.kreo.net/2d/projects)
  • Upload "Project Name - PF Bid Docs" to Kreo
  • Foundation Plan: Count column footings by type (EA), wall footings by type (LF), slab area by type (SF)
  • Civil Plan: Get SF of slabs to cross-check; get SF of site perimeter, convert to acres (SF / 43,560)
  • Plug Building Footprint (SF) into H17 and Site Size (Acres) into H18
Estimating
8
Receive Prelim Design
Download Dr. Ed's preliminary design and cross-check takeoffs.
  • Save prelim design in main project folder (not a subfolder)
  • Compare Dr. Ed's LF and column counts against your takeoffs
  • Only flag major discrepancies
  • Copy prelim sheets into Excel estimate as third tab (replacing stock tab)
  • Also copy "Design Notes" tab as the last tab in the estimate
Estimating
9
Get Vendor Quotes
Obtain quotes for stone material, transportation, and column layout.
  • Stone: Get 3 quotes for #57 stone (washed, no fines, 3/8"-1" chips). Start 1 week before bid due date.
  • Transport: Get quotes from both Paddacks (Miah) and Stephan (Mark Maller)
  • Transport from: 14308 Figel Rd, Monroeville, IN 46773 to project site
  • Loads: Cat 336 excavator/vibro (98k lb), 80k excavator, F450/F550 + trailer, fall-off testing materials
  • Layout: Email Brett Miller at MLS with column count for $/column quote ($8-14/column typical)
Estimating
10
Build Estimate
Plug all values into the Excel template estimate. Key cells auto-populate from prelim design.
  • H9: LF of piers (link to Prelim Design Summary, typically T20)
  • H15: Stone tons (link to Prelim Design Summary, typically T21)
  • H16: Column count (link to Prelim Design Summary, typically T19)
  • H17: Building SF / H18: Site acres
  • Engineering cost: From Dr. Ed prelim (typically cell W54)
  • Stone + delivery: Use cheapest quote, note supplier in cell notes
  • Travel days: Minimum 2, add 1 per 6 working days
  • Mileage: Farm to site x2 (round trip), multiply for longer projects
  • Mobilization: Use cheaper of Paddacks or Stephan, note in cell
  • Equipment rental: Minimum 1 week, add 1 week per 5 working days
Estimating
11
Review with Head of Estimating
Meeting to review the complete estimate and project details before proposal.
  • Review project total price at bottom of estimate
  • Discuss alternates (slab support, helical piles for additions)
  • Confirm spoils amount, layout alternate pricing, timeline
Head of Estimating
12
Build Proposal
Update the Word template proposal with project-specific information and pricing.
  • Update date, project name, project address
  • Update contract documents date (from bid set drawings)
  • Update architect of record and date
  • Update geotechnical engineer and report date
  • Plug in pricing as directed in bid review meeting
  • Spoils: 1.5x Total LF for 24" columns, 2x Total LF for 30" columns
  • Layout alternate: Quote from MLS x 25-50%
  • Duration: Estimated working days + 3-5 days, present as range
  • Update bearing pressure PSF from GGG prelim
  • Update tax inclusion/exclusion per project
Estimating
13
Submit Bid
PDF the proposal and send with the reviewed estimate for final approval, then to the GC.
  • PDF the Word document proposal
  • Send reviewed Excel estimate + PDF proposal to Head of Estimating for final sign-off
  • Upon approval, submit to GC

Project Award SOP

From Letter of Intent through crew mobilization

Award-to-Mobilization Timeline

Submittals to GGG3-4 weeks

Mobilize
1
Request Submittals from GC
Request all necessary shop drawing information from the awarding GC within 1 week of NTP/LOI.
  • Structural Foundation Plan - Current Set (CAD)
  • Civil Grading Plan - Current Set (CAD)
  • Full Structural - Current Set (PDF)
  • Full Civil - Current Set (PDF)
  • Design Service Loads (DL + LL)
  • Any additional geotechnical reports (post-bid date)
  • Site grade elevation during PF installation (within 3" +/-)
  • Check GC schedule for PF installation timing
Estimating / PM
2
Notify Dr. Ed (GGG)
Email Garbin Geostructural Group that the project was awarded. Provide PF project number.
  • Ask what additional information is needed beyond standard submittal items
  • Allow 3-4 weeks for engineered shop drawing preparation once all docs received
Estimating
3
Update Bid Log
Move the project in the bid log from Submitted to Awarded section.
  • Assign next sequential job number in Column B
  • Flip bid status in Column U to "Awarded"
  • Update bid total value in Column V to expected contract value
Estimating
4
Move Project Folders
Relocate project files from 03-Estimating to 04-Project Management.
  • Move folder to: 04 - Project Management > 02 - Projects
  • Rename folder: XX-XXX - Project Name - General Contractor
Estimating / PM
5
Set Up Template Folders
Copy template folder structure into the newly moved project folder.
  • Preconstruction
  • Project Management
  • Engineering & Design
  • GC Drawings & Specs
  • Field
  • Organize all existing documents into the correct subfolders
Estimating / PM
6
Update Project Schedule
Update the PF Project Master Sheet with schedule details.
  • Color code project by status (LOI, contract, etc.)
  • Plug in total LF and column count
  • Calculate working days: LF / 1000, mark out that many days
  • Plug in start date based on GC response
PM
7
Update Project Dashboard
Add the project to the PF Project Dashboard Tab in the Master Sheet.
  • Copy "TEMPLATE NEW JOB" columns and insert between most recent project and template
  • Fill out known project information immediately
  • Yellow highlighted items are project-specific and need updating
  • Check back daily for outstanding items; push for info by GC/GGG deadlines
PM
8
Manage Submittal Process
Coordinate document flow between GC and Garbin Geostructural Group (GGG).
  • Track all documents sent and received between GC and GGG
  • Coordinate between Estimating and PM on who tracks each item
  • Once GGG completes submittals, send to GC for Owner/Design/GC review
  • GC review typically 1-2 weeks
Estimating / PM
9
Create Turnover Budget
Convert the bid estimate into the project turnover budget using actual submittal values.
  • Open project estimate from Preconstruction folder
  • "Save As" YY-MMDD Project Name - Turnover Budget
  • Adjust to actual LF, columns, and stone tonnage from approved submittals
  • Spot check all line items with revised values
  • Adjust contingency so contract value matches turnover budget exactly
  • Confirm selected alternates match contract (pier layout, helical piles, etc.)
Estimating / PM
10
Project Turnover
Send finalized budget and project info to the project manager at least 1 week before start.
  • Create Project Contact Info Turnover Sheet from template (02-Project Management folder)
  • Name: YY-MMDD - Project # - Project Name - Project Info
  • Fill all yellow highlighted cells in contact sheet
  • Email turnover budget + project info sheets to PM
  • Note any special items for site team (stone credit apps, etc.)
  • Hold turnover meeting if high-risk or unusual items exist
Estimating / PM

1
Download Column Log Data
Download zip file with column logs from Operations Manager.
  • Save to: Project Folder > Engineering & Design > QAQC > Guhma Data
QA/QC
2
Open GUHMA in Extended Mode
Launch the GUHMA desktop application and load all data files.
  • Click Extended Mode
  • Click Open Files, navigate to saved data folder
  • Select all data files and click Open
QA/QC
3
Configure Report Settings
Set up document title, project info, and PF branding.
  • Click Report Settings (bottom left of window)
  • Fill in Project # and Project Title
  • Add PF Logo if not already selected
QA/QC
4
Set Up Time Diagram
Configure the Time Diagram graph parameters for the report.
  • Row 1: Select "2 depth" - min = next interval of 5 above deepest column, max = 0
  • Interval: 1 for under 20', 2 for over 20'
  • Row 2: Always None
  • Row 3: Select "7 Buckets" - min=0, max=5, interval=1
  • Rows 4, 5: Always None
QA/QC
5
Set Up Depth/Time Diagram
Configure the vertical Depth/Time Diagram with 5 data channels.
  • Diagram 1: "2 depth" - match Time Diagram settings, smoothing=0
  • Diagram 2: "14 Flot" - min=0, max=5000, interval=1000, smoothing=1
  • Diagram 3: "9 Winch A (Down)" - min=0, max=5000, interval=1000, smoothing=1
  • Diagram 4: "10 PSI B (Winch Up)" - min=0, max=5000, interval=1000, smoothing=1
  • Diagram 5: "7 Buckets" - match Time Diagram settings, smoothing=0
QA/QC
6
Configure Table, Printout, and Profile
Finalize output settings for the report.
  • Table tab: Uncheck "Show Table"
  • Printout tab: Match standard PF selections
  • Profil/Others tab: Check "Show profile"
  • Update Job Site Data via info button: Project name, area (project #), contractor (PF), client (GC), Equipment ID (VSC Rig)
QA/QC
7
Review Each Column Against Submittal
The core QC check: verify all columns are installed correctly per the approved design.
  • Print GI-300 column layout page to PDF, save in QAQC folder
  • Check 1: All columns accounted for? Compare GUHMA pile list to GI-300 submittal
  • Check 2: All columns hit design depth? Type depths on submittal page (blue = meets/exceeds, red = short)
  • Check 3: Buckets match hand logs? Cross-reference GUHMA data with field hand logs
  • Missing columns: Notify PF owner (Jonathan Reinking) immediately
  • Short columns: Flag red, send to Dr. Ed for engineering review
QA/QC
8
Fix Data Discrepancies
Edit raw data files in SharePoint to correct bucket counts that don't match hand logs.
  • Navigate to raw data file for the column in SharePoint
  • Open in Notepad, scroll to bottom of file
  • Bucket column is one column left of the ";0;" column
  • Update last 3-5 numbers to correct bucket count
  • Save file - GUHMA will reflect the update
QA/QC
9
Generate Combined PDF Report
Create the final GUHMA report for project closeout.
  • Select all piles: Click top column, hold Shift, click bottom column
  • Click "Create combined PDF with selected files"
  • Save to a new "GUHMA" folder, then copy to project QAQC folder
  • Report includes one page per column with all diagrams
  • This PDF is a closeout deliverable sent to GC and Dr. Ed
QA/QC
Section B: Modulus Testing
1
Copy Modulus Test Template
Get the template from SharePoint and prepare it for the specific project.
  • Source: SharePoint > 04 - Project Management > 03 - AP Modulus Test
  • Copy "AP Modulus Testing Report_PF"
  • Paste to: Project Folder > 03 - Engineering & Design > Modulus Test
  • Rename: YY-MMDD Project Name Modulus Test
QA/QC
2
Fill In Project Info
Enter basic project information in the grey fillable fields.
  • Job Name, Location, Number, Tested By
  • AP Number and Test Date: filled out during actual field test
QA/QC
3
Set Design Parameters from Submittal
Pull design values from Dr. Ed's approved submittal and enter into the test sheet.
  • AP Design Diameter: 2.00' (24") or 2.50' (30") - check GI-200 page
  • Diameter of Plate: Same as AP Design Diameter
  • AP Reaction Modulus: From GI-100, AP Design and Installation Criteria section
  • AP Design Load: From GI-200 Modulus Test Schedule, applied load column (kips)
  • Jack Calibration: Small jack (under 175 kips) = 49.84 / Large jack (over 175 kips) = 32.88
  • Jacks recalibrated annually by Chicago Jack, new factors from Dr. Ed/GGG
QA/QC
4
Verify and Hand Off
Confirm all calculated values match the submittal, then the sheet is ready for field use.
  • Confirm % design load, load increment, and kips from submittal match the Excel sheet
  • Rest of sheet auto-populates and is ready for field operator during test
